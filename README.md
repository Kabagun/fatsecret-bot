# fatsecret-bot

Telegram-бот для пары пользователей: показывает объединенный список рецептов из двух FatSecret аккаунтов и синхронизирует выбранный рецепт из одного аккаунта во второй.

## Что уже реализовано

- Два Telegram-пользователя:
  - если `TELEGRAM_ALLOWED_USER_IDS` задан, пускаются только эти id;
  - если не задан, первые два пользователя, вызвавшие `/start`, регистрируются автоматически.
- Два FatSecret аккаунта добавляются через Telegram-кнопку `Аккаунты`; логин/пароль проверяются перед сохранением.
- Импорт cookbook из обоих аккаунтов и merge по нормализованному названию рецепта.
- Просмотр общего списка `/recipes`.
- Поиск по общему списку рецептов.
- Обновление из FatSecret `/refresh`.
- Просмотр рецепта с ингредиентами.
- Синхронизация выбранного рецепта из выбранного FatSecret аккаунта во все подключенные аккаунты.
- Удаление рецепта из FatSecret на всех привязанных аккаунтах.
- Batch-удаление нескольких рецептов из FatSecret.

## Важное ограничение MVP

Создание рецепта на втором аккаунте подтверждено live-capture: `RecipeActionAndroidPage.aspx` с `action=recipeinitialsave`, `prid=0`; ответ имеет формат `SUCCESS:<recipe_id>`. Затем бот переносит ингредиенты через `ingredientsave` и сохраняет метаданные через `recipesave`.

Удаление ингредиентов в удаленном FatSecret рецепте пока не реализовано: в capture был только `ingredientsave`. Бот обновляет найденные ингредиенты через их `iid`, добавляет отсутствующие и показывает, сколько лишних ингредиентов осталось в целевом рецепте.

Удаление рецепта восстановлено из APK: `RecipeActionAndroidPage.aspx` с `action=recipedelete`, `rid=<recipe id>`, `fl=5`. Успешное удаление снимает remote-привязку аккаунта; локальная карточка удаляется только когда рецепт удален на всех привязанных аккаунтах.

Создание собственных продуктов FatSecret пока требует отдельный capture endpoint. Если перенос ингредиента не проходит из-за собственного продукта, бот явно пишет об этом в результате синхронизации.

## Установка

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
```

Заполнить `.env`:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_ALLOWED_USER_IDS` или оставить пустым для авто-регистрации первых двух пользователей
- при необходимости `FATSECRET_AUTHORIZATION` и `FATSECRET_C_DESC`

FatSecret логины/пароли вводятся в боте через `Аккаунты` и сохраняются в runtime SQLite. `.env` и runtime SQLite лежат вне git.

## Запуск

```powershell
.\.venv\Scripts\Activate.ps1
python run_bot.py
```

Или после установки пакета в editable-режиме:

```powershell
pip install -e .
fatsecret-bot
```

## Команды

- `/start` - зарегистрироваться/проверить доступ.
- `/refresh` - подтянуть cookbook из двух FatSecret аккаунтов и смержить список.
- `/recipes` - показать объединенный список рецептов.
- `/search` - поиск по объединенному списку рецептов.

В карточке рецепта есть кнопки:

- `Синхронизировать`
- `Удалить в FatSecret`
- `К списку`

После `/start` бот показывает постоянные кнопки:

- `Рецепты`
- `Поиск`
- `Обновить`
- `Аккаунты`

В списке рецептов есть пагинация и кнопка `Удалить несколько` для batch-удаления. Рецепты создаются и редактируются в FatSecret, затем подтягиваются кнопкой `Обновить` и синхронизируются через карточку рецепта.
